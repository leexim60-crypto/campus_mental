# 校园心理健康智能评估与干预平台 · 三人分工说明

本文档与当前仓库实现一致：**后端 FastAPI + SQLAlchemy**，**前端 Vue 3 + Vite**，接口前缀 **`/api/v1/`**。三人按业务线划分，减少同时修改同一模块的冲突。

---

## 一、通用约定（三人共同遵守）

1. **接口规范**：统一返回 `{ code, msg, data }`；需要登录的接口使用 `Authorization: Bearer <token>`。
2. **数据库**：表结构以 `backend/main.py` 中 SQLAlchemy 模型为准；若需改表，**先群内确认再改**，避免联调断链。
3. **协作流程**：按模块建分支或提 PR；改公共文件（如 `App.vue`、`router/index.js`、`api/index.js`、`main.py` 中非本人前缀路由）时，在 PR/群消息里 **@相关成员** 说明变更点。
4. **自测**：后端可用 `http://127.0.0.1:8002/docs`（Swagger）或 Postman；前端 `npm run dev` 联调代理 `/api` → 8002。

---

## 二、成员 A：学生账号、测评与预约模块
负责 **学生侧身份** 与 **心理测评全流程**（选题、答题、算分、结果展示、历史记录），测评结果中 **AI 点评** 与后端 LLM 调用的衔接（与 `llm_client.py` 中测评相关逻辑对接），以及 **咨询预约**（提交、我的预约、取消）。

### 2.2 后端（建议负责范围）

| 前缀/模块 | 说明 |
|-----------|------|
| `POST /api/v1/user/login` | 学生登录 |
| `POST /api/v1/user/register` | 学生注册 |
| `POST /api/v1/user/reset-password` | 重置密码 |
| `GET /api/v1/evaluation/get-questions` | 获取量表题目 |
| `POST /api/v1/evaluation/calculate-result`（及同类提交结果接口） | 计算并保存测评结果 |
| `GET /api/v1/evaluation/get-my-results` | 我的测评列表 |
| `GET /api/v1/evaluation/result-detail` | 测评详情 |
| 测评链路中的 LLM 调用 | 与 `llm_client.py`、环境变量配置协同 |
| `POST /api/v1/appointment/*`、`GET .../my-list` | 预约提交、列表、取消 |

关联数据表：`user`（学生角色）、`evaluation_question`、`evaluation_result`、`counseling_appointment`。

### 2.3 前端（建议负责范围）

| 路径/文件 | 说明 |
|-----------|------|
| `src/views/LoginView.vue` | 学生登录、找回密码入口 |
| `src/views/RegisterView.vue` | 注册 |
| `src/views/EvaluationView.vue` | 测评答题 |
| `src/views/ResultView.vue` | 结果展示 |
| `src/views/AppointmentView.vue` | 心理咨询预约 |
| `src/stores/studentAuth.js` | 学生登录态 |
| 路由 `meta.requiresAuth` 中学生相关守卫 | 与 B/C 协调，避免重复逻辑 |

## 三、成员 B：管理员、数据统计与资源库模块

### 3.1 职责概述

负责 **管理员登录与鉴权**、**全校/全库测评数据统计**、**测评记录管理**、**数据导出**、**可视化大屏**、**管理员个人中心**（资料展示、修改密码等），以及 **心理资源库**（列表、详情、种子内容维护）。

### 3.2 后端（建议负责范围）

| 前缀/模块 | 说明 |
|-----------|------|
| `POST /api/v1/admin/login` | 管理员登录 |
| `POST /api/v1/admin/register` | 受控自助注册（需配置 `ADMIN_REGISTER_SECRET`） |
| `GET /api/v1/admin/check-permission` | 权限校验 |
| `GET /api/v1/admin/info` | 管理员信息 |
| `POST /api/v1/admin/change-password` | 管理员改密 |
| `GET /api/v1/admin/statistic/*` | 情绪、量表等统计 |
| `GET/POST /api/v1/admin/export-*` | 导出 CSV 等 |
| `GET /api/v1/admin/evaluation/list` | 管理员侧测评列表 |
| `GET /api/v1/resource/list`、`detail` 等 | 资源列表与详情 |

关联数据表：主要读写 `user`（`role=admin`）、`evaluation_result`；统计聚合可联表 `user`；`mental_resource` 种子内容由 B 维护。

### 3.3 前端（建议负责范围）

| 路径/文件 | 说明 |
|-----------|------|
| `src/views/AdminLoginView.vue` | 管理员登录 |
| `src/views/AdminStatisticView.vue` | 统计图表、导出 |
| `src/views/AdminEvaluationView.vue` | 测评记录管理 |
| `src/views/AdminDashboardView.vue` | 数据大屏 |
| `src/views/AdminPersonalCenterView.vue` | 管理员中心 |
| `src/views/ResourceListView.vue`、`ResourceDetailView.vue` | 心理资源库 |
| `src/stores/adminAuth.js` | 管理员登录态 |
| 路由 `meta.requiresAdmin` | 管理员路由守卫 |

### 4.1 职责概述

### 4.2 后端（建议负责范围）

| `POST /api/v1/ai/chat` | 心灵树洞多轮对话 |

关联数据表：读 `user`、`evaluation_result`、`counseling_appointment`（个人中心展示）。

### 4.3 前端（建议负责范围）

| 路径/文件 | 说明 |
|-----------|------|
| `src/views/PersonalCenterView.vue` | 学生个人中心 |
| `src/views/AiChatView.vue` | 心灵树洞 |

## 五、公共与交叉事项

| 内容 | 建议 |
|------|------|
| `backend/main.py` | 按路由前缀归属 A/B/C；合并前解决冲突；模型定义变更需三人知情。 |
| `backend/config.py`、`security.py`、`llm_client.py` | 配置与安全属公共；**LLM 行为** 测评侧偏 A，对话侧偏 C，改前互相知会。 |
| `src/App.vue`、`src/router/index.js` | 公共导航与守卫；改动时简短说明，避免覆盖他人入口。 |
| `src/api/index.js` | Token 拦截规则；管理员/学生 URL 分支勿随意改。 |
| 数据库首次部署 | 空库启动后端会自动建表；**MySQL 需先建空库**（见 README）。 |

## 六、验收清单（可按周报勾选）

- **A**：学生可注册/登录/测评/看结果与历史；测评 AI（若配置）可用；可预约/取消咨询。
- **B**：管理员可登录；统计、列表、导出、大屏正常；管理员中心可用；资源库可浏览。
- **C**：个人中心数据齐；心灵树洞在 Ollama/DeepSeek 配置下可对话。
