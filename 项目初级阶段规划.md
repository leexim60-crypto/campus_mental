# 校园心理健康智能评估与干预平台 · 初级阶段规划

> 本文档描述项目的 **背景与要解决的问题**、**初级阶段**（需求对齐、架构搭建、核心功能开发与联调）的 **开发内容、目标、计划、技术选型与人员分工**，适用于课程大作业、毕设开题与中期检查等材料引用。  
> **详细接口与文件级分工**见 [分工说明.md](./分工说明.md)。

## 一、背景介绍

### 1.1 现实问题

大学生在学习、人际与未来发展等方面承受的压力普遍增加，**情绪困扰与心理亚健康**并不罕见。常见痛点包括：

- **求助入口不统一**：宣传材料分散在通知、公众号与海报上，学生在情绪低落时**不知道去哪预约、先看什么资料**，容易拖延或放弃求助。  
- **筛查与预防手段偏传统**：仅靠线下讲座或人工摸排，**覆盖面有限**，也难以及时掌握群体情绪与量表使用的大致分布。  
- **即时倾诉渠道不足**：部分同学需要**低门槛、可匿名感更强**的文字倾诉，而完全依赖真人咨询师在时间与人力上难以满足即时性。  
- **管理侧缺少数据支撑**：心理中心或学工若仍靠 Excel 汇总，**统计效率低**，不利于开展针对性的宣传与干预资源配置。

### 1.2 本项目要解决的问题

本系统面向 **校园场景**，用一套 **Web 应用** 把「自助筛查—科普与资源—预约衔接—轻量 AI 陪伴—管理侧统计」串成闭环，目标包括：

| 面向对象 | 解决的问题 |
|------------|------------|
| **学生** | 统一入口完成 **心理量表自评**、查看结果与历史；按需浏览 **心理科普资源**；提交 **咨询预约**；在需要时使用 **心灵树洞** 与 AI 进行多轮文字倾诉（**不替代**专业咨询，并在提示中强调边界）。 |
| **管理员 / 心理工作相关老师** | 通过 **统计图表、测评记录查询、数据导出与大屏**，了解量表使用与情绪标签分布等情况，为宣传、讲座与资源配置提供 **可视化依据**（本阶段为演示级统计，非临床诊断系统）。 |

### 1.3 定位说明

本系统定位为 **心理健康教育与自助支持工具**，量表结果用于**筛查与自我了解**，**不能替代**医疗机构诊断或面对面心理咨询。初级阶段优先保证 **流程跑通、可演示、可扩展**，与《分工说明》中的模块划分一致。

## 二、阶段定位

初级阶段聚焦：**可运行的前后端分离系统**、**学生与管理员两条业务闭环**、**本地可演示的 AI 能力（Ollama / DeepSeek）**，不追求大规模并发与生产级运维（可放在后续阶段）。

---

## 三、开发内容（本阶段交付范围）

| 模块 | 内容概要 |
|------|-----------|
| **学生与认证** | 注册、登录、重置密码；JWT；个人中心基本信息与测评/预约汇总 |
| **心理测评** | PHQ-9、SCL-90（演示）量表；答题、计分、结果页；历史记录与详情；可选 AI 长文点评 |
| **咨询预约** | 提交预约、我的预约列表、取消（规则内） |
| **心理资源库** | 资源列表、筛选、详情（文章/音频类） |
| **心灵树洞** | 多轮对话；流式输出（SSE）；依赖本机 Ollama 或云端 DeepSeek |
| **管理端** | 管理员登录；受控自助注册（`ADMIN_REGISTER_SECRET`）；统计图表、测评记录管理、CSV 导出、数据大屏 |
| **公共** | 统一接口前缀 `/api/v1/` 与 `{ code, msg, data }`；环境变量与 `.env.example`；README 与分工文档 |

**本阶段明确不包含（可列为后续）**：短信/邮件验证码、支付、线上心理咨询视频、多校租户、完整等保测评、Kubernetes 部署等。

## 四、阶段目标（验收口径）

1. **功能**：学生可完成注册—测评—看结果—预约—浏览资源—使用心灵树洞；管理员可登录（或受控注册）后查看统计与管理测评数据。  
2. **数据**：MySQL（或开发用 SQLite 若团队另有约定）中表结构一致，关键业务有基本种子数据可演示。  
3. **安全**：密码 bcrypt；接口按角色鉴权；敏感配置不进仓库。  
4. **AI**：在配置 Ollama 或 DeepSeek 的前提下，测评点评与树洞对话可用；未配置时有合理降级或提示。  
5. **文档**：环境与启动步骤可复现；分工与接口约定清晰，第三方可据 README 跑通主流程。

## 六、使用技术

### 6.1 技术栈总览

| 层次 | 技术 | 说明 |
|------|------|------|
| **后端语言与框架** | Python 3.10+、FastAPI | REST API、依赖注入、OpenAPI（Swagger） |
| **ORM 与数据库** | SQLAlchemy 2.x、MySQL（PyMySQL） | 模型与迁移思路以代码与 README 为准 |
| **安全** | bcrypt、python-jose（JWT）、python-dotenv | 密码哈希、Bearer Token、本地配置 |
| **AI** | Ollama（本机）、DeepSeek API（可选）、httpx | 测评点评与树洞；流式 SSE |
| **前端框架** | Vue 3（`<script setup>`）、Vite | 单页应用与热更新 |
| **前端状态与路由** | Pinia、Vue Router 4 | 学生/管理员双登录态、路由守卫 |
| **UI 与样式** | Element Plus、Tailwind CSS v4 | 组件库与工具类样式 |
| **数据可视化** | ECharts | 管理端统计图与数据大屏 |
| **HTTP 客户端** | Axios（常规请求）、Fetch（SSE 流式） | 与 Vite 代理 `/api` → 后端 |

### 6.2 环境与工具

- **Node.js** 18+、**npm**；**Python** 3.10+（推荐 3.11，避免过新版本缺 wheel）。  
- **Git** 版本管理；可选 **Postman** / Swagger UI 做接口自测。  
- 可选 **Docker / Compose**（若课程要求容器化，可在下一阶段单独说明）。

## 八、风险与依赖（初级阶段）

| 风险 / 依赖 | 应对思路 |
|-------------|----------|
| MySQL 环境不一致 | 统一 `DATABASE_URL` 写法；README 写明建库字符集 `utf8mb4` |
| Python 依赖安装慢或失败 | 固定 3.11；`pip` 升级；必要时国内镜像 |
| Ollama 仅本机可用 | README 说明 `127.0.0.1` 与 WSL/Docker 场景；演示可改用 DeepSeek |
| 合并冲突集中在 `main.py` / `App.vue` | 按路由前缀与导航分区认领；小步合并 |
